## Reflexion

A la hora de realizar el TP, arranque desde la base que tenía del TP6 realizado anteriormente, donde tuve que entender que cambiaba al misgrar a React Native (no hay HTML, no hay CSS, no hay OnClick, no hay stopPropagation, y más cosas que tuve que ir aprendiendo a la hora de migrarlo). Al realizar el proyecto, tuve algunas dificultades de como hacerlo, entonces me apoyé de la IA usando Claude Code para que me explique como encarar el TP8 y poder llegar al objetivo que se pedía. Una vez que entendía como hacerlo, empecé a desarrollarlo con mis conocimientos, mientras también usaba la IA y las skills para poder llegar a un mejor resultado.
Siempre mantuve el mismo figma de referencia para llegar al resultado, donde cambié un poco la estructura del proyecto comparado a como era en formato web. Al principio el código quedó muy complejo — tenía Context API, tres pantallas distintas en el Stack Navigator y un archivo separado solo para la navegación. Me di cuenta que era demasiado para lo que pedía la consigna, así que decidí simplificarlo: eliminé el Context API y volví al mismo patrón que usé en el proyecto web, con todo el estado centralizado en App.tsx. Para el detalle del post usé un Modal nativo de React Native en vez de una pantalla separada, lo que evitó el problema de tener el mismo estado duplicado en dos lugares distintos.
Algo que aprendí en este proyecto fue la diferencia entre usar .map() y FlatList para renderizar listas. En el proyecto web usaba .map() directamente en el JSX para mostrar los posts, pero en este TP la consigna lo prohíbe explícitamente para el feed. Entendí por qué: si tenés 1000 posts, .map() los renderiza todos de una sola vez aunque el usuario solo vea 3 en pantalla, lo que consume memoria y hace la app lenta. FlatList en cambio solo renderiza los elementos que están visibles en pantalla en ese momento, y va cargando el resto a medida que scrolleás. Para este proyecto con 12 posts no se nota la diferencia, pero es una buena práctica que escala bien si la app creciera.
Lo que más me gustó de este proyecto es que ya se tenía una base para poder migrar a React Native
